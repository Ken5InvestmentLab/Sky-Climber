# Repository Guidance

## Project Overview

- This is a Vite + React browser game.
- The primary app and game logic live in `src/App.jsx`.
- Static assets live in `public/`; `dist/` is generated by the deploy build.

## Build and Test

- Run `npm run build` to verify production builds.
- There is no standalone test runner yet; helper-level checks live in `runSelfTests()` in `src/App.jsx` and run when the app mounts.
- When changing reusable game logic, score logic, or date logic, add focused assertions to `runSelfTests()`.

## Score and Leaderboard Rules

- Monthly score behavior is based on JST month boundaries via `getMonthKey()`.
- Keep local high score storage keys and Supabase leaderboard `month_key` aligned with the same JST month key.
- Avoid reusing all-time local score keys for monthly ranking or high score display.

## Generated Files

- Do not hand-edit `dist/`; regenerate it with `npm run build` when a deployment artifact is intentionally needed.
